## Chain definition language Chainblocks is more oriented to be a meta language or a visual language rather then a pure language but to nicely describe chains and the data around it we adopt and write a simple language based on EDN and Clojure. You can basically think of it as a comprehensible json on steroids that can be evaluated. The exection of each of these programs is meant to be fast, no run loops within. Basically load, define the chain and return a ready to run node from it. ### Example ``` lisp (Chain :Name "Test Chain 1" :Looped true ("Hello world!") (Log)) (Chain :Name "Test Chain 2" (3.14) (Set 'x) (2) (Math.Multiply 'x) (Log)) (def mul Math.Multiply) (def BadPI (3.14)) (defn chainme [a] (Chain :Name (str "Test Chain " a) BadPI (Set 'x) (a) (mul 'x) (Log))) (chainme 3) (chainme 5) ``` ### Core keywords ##### `def` * They are always global definitions. ##### `defn` ##### `defmacro` *TODO* ##### `let` ##### `if` ##### `do` ##### `str` ##### `seq` ##### `conj` ##### `cons` ##### `concat` ##### `nth` ##### `first` ##### `rest` ##### `apply` ##### `map` ### Operators #### Math ##### `+` ##### `-` ##### `*` ##### `/` ##### `mod` #### Logic ##### `=` ##### `not` ##### `<` ##### `>` ##### `>=` ##### `>=` ##### `and` ##### `or`
